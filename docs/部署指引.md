# 部署指引

> 最后更新：2026-06-07  
> 目标：将**只读** Wiki 服务（浏览、搜索、Chat）暴露给用户；**Ingest 仍在本地**。

---

## 1. 部署什么、不部署什么

| 组件 | 本地 | 服务器 |
|------|------|--------|
| Ingest / 改 wiki | ✅ | ❌ |
| `wiki/` 数据 | 源 | rsync 只读副本 |
| `llm-wiki-server` + UI | 可选 | ✅ |
| Node.js（Chat 子进程） | ✅ | ✅ **必需** |
| LLM / Embedding API Key | 配置在 server | 环境变量或 config |

---

## 2. 方案选型

| 方案 | 适用 | 文档 |
|------|------|------|
| **低配 ECS 一键脚本**（推荐小内存机） | 服务器 ≤ 2GB RAM 无法本地编译；本地 musl 静态 + 一键 rsync | [部署-低配ECS一键脚本](./部署-低配ECS一键脚本.md) |
| **ECS + Cloudflare Tunnel**（推荐通用） | 国内用户、免开放 8080、HTTPS | [部署-ECS与Tunnel](./部署-ECS与Tunnel.md) |
| **Docker Compose** | 快速本地/内网验证 | [docker/README.md](../docker/README.md) |
| **海外 VPS + Tunnel** | 国际用户、低延迟访问 OpenAI 等 | [部署-ECS与Tunnel §5](./部署-ECS与Tunnel.md#5-阶段二迁到海外-vps) |

Cloudflare Tunnel 使用 **cloudflared**，不是 SSH 反向代理。

---

## 3. 部署前检查清单

- [ ] 本地 wiki 已 Ingest，搜索/Chat 在 `127.0.0.1:8080` 验证通过  
- [ ] `VITE_API_TOKEN` 与 `LLM_WIKI_API_TOKEN` 一致并已写入构建产物  
- [ ] `server.local.json` 中 `projects[].path` 指向服务器上的 wiki 路径（如 `/data/wiki/ParentingBooks`）  
- [ ] Chat：`LLM_WIKI_REPO`、`upstream/src/`、`overlay/cli/node` + `npm ci` 已就绪（见 [部署-ECS与Tunnel §3.5.1](./部署-ECS与Tunnel.md#351-chat-运行时依赖说明)）  
- [ ] systemd 使用 `--bind 127.0.0.1:8080`，安全组不开放 8080  
- [ ] 密钥文件权限 `600`，不进 Git  

---

## 4. 上线后更新 wiki

```bash
rsync -avz --delete \
  ~/path/to/llm_wiki_projects/ParentingBooks/ \
  deploy@ECS:/data/wiki/ParentingBooks/
```

程序更新：重新构建 `llm-wiki-server` + `upstream/dist`，rsync 到 `/opt/llm-wiki/`，`systemctl restart llm-wiki-server`。

日常流程见 [日常运维 §7](./日常运维.md#7-同步到线上已部署时)。

## 5. 已知限制

| 项目 | 说明 |
|------|------|
| Docker 镜像 | 默认**不含 Node**，Chat 需宿主机 Node 或改镜像 |
| HTTP 搜索 | 关键词为主；向量混合搜索待后续 |
| HTTP 写入 | 不支持 ingest / 改 wiki |
| Token | 构建时嵌入 UI bundle，轮换 token 需重建 UI |
