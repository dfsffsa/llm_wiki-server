# 备份与恢复

> 商用前必须落地的数据保护。本仓库的 auth SQLite（用户/会话/对话/用量）和 wiki 数据都需要备份。

## 1. 需要备份什么

| 数据 | 位置 | 重要性 | 频率 |
|------|------|--------|------|
| auth SQLite | `$LLM_WIKI_AUTH_DB`（如 `/data/llm-wiki/auth.db`） | 高（用户账号、对话历史、用量） | 每日 |
| wiki 内容 | `$LLM_WIKI_PROJECT/wiki/`、`raw/sources/` | 高（知识库本身） | 每次 ingest 后 |
| LanceDB 向量索引 | `$LLM_WIKI_PROJECT/.llm-wiki/lancedb/` | 中（可 `reindex --vectors` 重建） | 可不备份，重建即可 |
| 配置 | `server.local.json`（含 LLM_API_KEY） | 高 | 变更时 |

## 2. auth DB 热备份

[`scripts/backup-auth-db.sh`](../scripts/backup-auth-db.sh) 用 `sqlite3 .backup` 做在线热备——服务运行中可安全执行（SQLite backup API，非裸文件拷贝，产出事务一致快照），无需停服。

```bash
# 一次性
./scripts/backup-auth-db.sh /data/llm-wiki/auth.db /var/backups/llm-wiki 14
#   arg1: auth.db 路径（或 AUTH_DB env）
#   arg2: 备份目录（或 BACKUP_DIR env）
#   arg3: 保留份数（或 KEEP_COPIES env，默认 14）
```

产物：`<backup-dir>/auth-<UTC时间戳>.db.gz`，每次备份后自动 `PRAGMA integrity_check` 校验，失败即删除该份并告警；超过 `KEEP_COPIES` 的旧备份自动清理。

**依赖**：`sqlite3` CLI（`apt install sqlite3`）。

### cron（每日 03:17）

```cron
17 3 * * *  /opt/llm-wiki-server/scripts/backup-auth-db.sh /data/llm-wiki/auth.db /var/backups/llm-wiki 14 >> /var/log/llm-wiki-backup.log 2>&1
```

> 时间错开整点（17 分），避开与 ingest / 其他定时任务撞车。

## 3. 恢复 auth DB

```bash
systemctl stop llm-wiki-server
cp /var/backups/llm-wiki/auth-2026-06-26.db.gz /tmp/   # 选最近一份
gunzip /tmp/auth-2026-06-26.db.gz
cp /tmp/auth-2026-06-26.db /data/llm-wiki/auth.db
systemctl start llm-wiki-server
curl -sS http://127.0.0.1:8080/api/v1/health   # 确认 ok:true
```

注意：恢复到旧快照后，该快照之后注册的用户、新会话、新对话会丢失；已发出的 session cookie 会失效（用户需重新登录）。

## 4. wiki 数据备份

wiki 是文件（`.md` + `lancedb/`），用 rsync 到异地 + 保留策略：

```bash
# 每日异地同步（带日期快照，保留 30 天）
DEST=user@backup-host:/backups/llm-wiki/wiki-$(date -u +%Y-%m-%d)
rsync -a --delete /data/wiki/ "$DEST/"

# 清理 30 天前的快照（在 backup-host 上）
find /backups/llm-wiki/ -maxdepth 1 -name 'wiki-*' -mtime +30 -exec rm -rf {} +
```

向量索引（`.llm-wiki/lancedb/`）可不备份——丢失后用 `llm-wiki reindex --vectors` 重建即可，无需从备份恢复。

## 5. 配置备份

`server.local.json` 含 `LLM_API_KEY`，敏感。备份到受限目录：

```bash
cp /etc/llm-wiki/server.local.json /var/backups/llm-wiki/server.local.json.$(date -u +%Y-%m-%d)
chmod 600 /var/backups/llm-wiki/server.local.json.*
```

## 6. 恢复演练

商用前至少做一次完整演练：
1. 在测试机部署 + ingest 一批数据 + 注册几个用户 + 产生对话
2. 跑备份脚本
3. **删除** auth.db 和 wiki 目录（模拟灾难）
4. 按本文档恢复
5. 验证：用户能登录、对话历史在、wiki 搜索正常

不演练的备份等于没有备份——只有演练过才知道备份是否真的可用。
