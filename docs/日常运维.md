# 日常运维

> 最后更新：2026-06-07  
> 适用：已有 Wiki 项目（如 `ParentingBooks`、`CivilCareer`）的**日常维护**  
> 从零建新库见 [新项目指引](./新项目指引.md)

本文档回答：**如何添加材料、Ingest、Embedding、切换项目、本地验证、同步到线上**。

---

## 1. Wiki 数据在哪里

| 位置 | 内容 |
|------|------|
| `~/overseas-github/llm_wiki_projects/<项目名>/` | Wiki 项目根（**不进 Git**） |
| `raw/sources/` | 原始材料（只读，Ingest 输入） |
| `wiki/` | LLM 编译后的知识页（搜索与 Chat 的数据源） |
| `.llm-wiki/lancedb/` | 向量索引（`reindex --vectors` 后） |
| `purpose.md` / `schema.md` | 项目目标与页面结构规则（Ingest 时 LLM 阅读） |

集成仓库 `llm_wiki-server` 只放**程序与配置样例**；知识内容在项目目录。

---

## 2. 添加或更新材料

### 2.1 新增章节 / 文本

1. 将 Markdown、txt 等放入 `<项目>/raw/sources/`（文件名建议可读，如 `12-辅食添加.md`）。
2. PDF/Office：headless CLI 需桌面 pdfium，或先用 `preprocess --copy-fallback` 转成文本（见 [overlay/cli/README.md](../overlay/cli/README.md)）。
3. 若新材料类型与现有 `schema.md` 差异大，先更新 `purpose.md` / `schema.md` 再 Ingest。

### 2.2 手动改 wiki/（可选）

- **HTTP 服务**不支持写入 wiki；改文件请直接在项目目录编辑，或用**桌面版** upstream。
- 大改结构后建议重新 `reindex --vectors`（若启用 embedding）。
- 小改单页通常不必重新 Ingest 整本书。

### 2.3 更新已入库源文件

若 `raw/sources/foo.md` 内容变了，且 `wiki/sources/foo.md` 已存在：

- `ingest-batch.sh` 会 **SKIP** 已入库文件。
- 需**删除**对应 `wiki/sources/`（及相关 concepts/entities 若需重建）后，再单独 `ingest` 该文件。

## 3. Ingest（编译 wiki）

> **实现原理**（调用链、代码文件、`autoIngest` 各阶段）：见 [代码结构总览 §7.4](./代码结构总览.md#74-ingest-实现说明)。

### 3.1 一次性准备

```bash
cd ~/overseas-github/llm_wiki-server
./scripts/build-cli.sh   # 首次或 CLI 代码更新后

export LLM_WIKI_PROJECT=~/overseas-github/llm_wiki_projects/ParentingBooks
cp overlay/config/llm.example.json overlay/config/llm.local.json   # 或复用 server.*.local.json
export LLM_WIKI_CONFIG=overlay/config/llm.local.json
# 编辑 config：llmConfig.apiKey 或 ${LLM_API_KEY}
export LLM_API_KEY='...'
```

### 3.2 单文件试跑

```bash
./scripts/llm-wiki ingest \
  "$LLM_WIKI_PROJECT/raw/sources/01-示例.md" \
  --project "$LLM_WIKI_PROJECT" \
  --config "$LLM_WIKI_CONFIG"
```

成功标志：`[ingest] done — N wiki file(s) written`，并检查 `wiki/sources/`、`wiki/concepts/`。

### 3.3 批量 Ingest

```bash
./scripts/ingest-batch.sh
# 或后台：nohup ./scripts/ingest-batch.sh >> /tmp/ingest.log 2>&1 &
```

| 行为 | 说明 |
|------|------|
| 遍历 | `raw/sources/*.md` 逐个 ingest |
| SKIP | `wiki/sources/<同名>` 已存在则跳过 |
| 失败即停 | 任一篇失败脚本 `exit 1`；修好后重跑，已 SKIP 的不重做 |
| 日志 | 默认 `/tmp/llm-wiki-ingest-<项目名>.log` |

进度与排错详见 [新项目指引 §4.3](./新项目指引.md#43-批量-ingest)。

## 4. Embedding / 向量索引

HTTP **关键词搜索**不依赖 LanceDB；若 ingest 时或之后需要**向量索引**（桌面混合搜索、后续 HTTP 向量能力）：

1. 在配置中启用 `embeddingConfig`（与 ingest 时一致，如 DashScope `text-embedding-v4`）。
2. 执行：

```bash
./scripts/llm-wiki reindex --vectors \
  --project "$LLM_WIKI_PROJECT" \
  --config "$LLM_WIKI_CONFIG"
```

向量写入 `<项目>/.llm-wiki/lancedb/`。**同步到 ECS 时记得 rsync 整个项目目录或至少包含 `.llm-wiki/`**（若线上需要）。

> **限制**：当前 HTTP server 搜索以关键词为主；LanceDB 混合搜索待后续抽取 core crate。见 [开发与测试 §1](./开发与测试.md#1-开发进度phase-04)。

## 5. 切换项目

### 5.1 CLI（单项目）

```bash
export LLM_WIKI_PROJECT=~/overseas-github/llm_wiki_projects/CivilCareer
# 后续 ingest / search / reindex 均带 --project 或依赖此 env
```

### 5.2 HTTP 服务（多项目）

编辑 `server.local.json`（或 `LLM_WIKI_CONFIG` 指向的文件）：

```json
{
  "projects": [
    { "path": "/path/to/ParentingBooks", "current": true },
    { "path": "/path/to/CivilCareer" }
  ]
}
```

- `current: true` 标记默认项目（完整 HTTP UI 打开时使用）。
- 也可用 env：`LLM_WIKI_PROJECTS=/path/a:/path/b`，`LLM_WIKI_PROJECT=/path/a` 指定 current。
- **Lite UI**（`/lite/`）：首页卡片列出全部 `projects[]`，用户点选切换。

配置样例见 [overlay/config/README.md](../overlay/config/README.md)。

### 5.3 构建 UI 时的项目

- 完整 UI：运行时从 server 拉项目列表，一般**无需**构建时指定项目。
- 可选 `VITE_WIKI_PROJECT` 固定默认项（少见）。

## 6. 本地验证（浏览 / 搜索 / Chat）

```bash
# 1. 构建 UI（token 与 server 一致）
VITE_BACKEND=http VITE_API_TOKEN=your-token ./scripts/build-web.sh

# 2. 启动 server
export LLM_WIKI_STATIC=upstream/dist
export LLM_WIKI_CONFIG=overlay/config/server.minimax.local.json
export LLM_WIKI_API_TOKEN=your-token
export LLM_WIKI_REPO=$PWD
./overlay/server/target/release/llm-wiki-server --bind 127.0.0.1:8080
```

| 地址 | 用途 |
|------|------|
| `http://127.0.0.1:8080/` | 完整 HTTP UI |
| `http://127.0.0.1:8080/lite/` | Lite 问答 |
| `http://127.0.0.1:8080/api/v1/projects` | 项目列表（需 Bearer token） |

Chat / Token / 401 排错：[开发与测试 §Q5–Q8](./开发与测试.md#7-常见问题与答复faq)。

## 7. 同步到线上（已部署时）

Ingest 与 reindex **在本地完成**；线上 ECS 只读挂载 wiki。

```bash
rsync -avz --delete \
  ~/overseas-github/llm_wiki_projects/ParentingBooks/ \
  deploy@YOUR_ECS:/data/wiki/ParentingBooks/
```

- 含 `.llm-wiki/` 若线上依赖 LanceDB。
- 一般**无需重启** server；异常时再 `systemctl restart llm-wiki-server`。
- 完整部署流程见 [部署指引](./部署指引.md)。

## 9. 常见问题

| 现象 | 处理 |
|------|------|
| batch 中途 FAILED | 看 log 中该篇错误，修复后重跑 |
| batch 全 SKIP 但内容旧 | 删对应 `wiki/sources/` 再 ingest |
| HTTP 搜索无结果 | 确认 server `projects[].path` 指向正确 wiki；embedding 配置与 ingest 一致 |
| Chat 401 | `VITE_API_TOKEN` 与 `LLM_WIKI_API_TOKEN` 不一致，重建 UI |
| 想 GUI 入库 | 用桌面版 `upstream` + `tauri dev`（HTTP 不支持写入） |
